How To Use Agent Brain

Get your agents remembering in 3 steps.

1 Create a Thought (Project)

A thought is a container for one project. It gets its own access key so your agent can read/write only its own data.

curl -X POST https://YOUR-DOMAIN/api/brain \ -H "Authorization: Bearer bt_YOUR-BRAIN-TOKEN" \ -H "Content-Type: application/json" \ -d '{"thoughtId": "my-project", "description": "My cool project"}' # Response: # {"thoughtId":"my-project","key":"tk_a1b2c3...","message":"Thought created."}

Or use the Dashboard above to create thoughts with a click.

2 Give Your Agent the Thought Key

Paste this into your agent's system prompt or custom instructions:

You have a memory API. Before doing anything, read your memory: curl "https://YOUR-DOMAIN/api/brain/PROJECT-NAME/context" \ -H "Authorization: Bearer tk_YOUR-THOUGHT-KEY" This returns every message, decision, and action from all past sessions. Read it fully. Continue from where the last session left off. At the end of this session, save everything: curl -X POST "https://YOUR-DOMAIN/api/brain/PROJECT-NAME/sync" \ -H "Authorization: Bearer tk_YOUR-THOUGHT-KEY" \ -H "Content-Type: application/json" \ -d '{ "sessionId": "session-XXX", "messages": [ ...all messages... ], "planning": [ ...every thought and decision... ], "actions": [ ...files changed, commands run... ] }'

3 That's It

Your agent will now read its full history before starting work and save everything when it's done. Next session, it picks up exactly where it left off.
